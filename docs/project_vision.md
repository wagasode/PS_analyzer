# PS_analyzer プロジェクトビジョン

## この文書の位置づけ

この文書は、PS_analyzer の長期的な上位方針を継続的に参照・更新するための living document です。

Cloudflare Access 移行や戦績管理ページのような現在の重点テーマだけを説明する一時資料ではありません。機能追加、公開範囲、データ管理、運用方針を判断するときは、この文書をPS_analyzer全体の方向性を確認する基準として扱います。

大きな方針変更、新しい機能領域の追加、やらないことの見直しが発生した場合は、該当する issue や PR と合わせてこの文書も更新します。

## 目的

PS_analyzer は、Shadowverse: Worlds Beyond Premier Series に関する配信、デッキ、戦績の情報をチーム内で継続的に確認するための分析基盤です。

単発の調査メモや場当たり的な集計ではなく、同じデータを同じ観点で見続けられる状態を作り、練習、対策、振り返り、意思決定を支援することを目的とします。

## 主な利用者

- チームメンバー
- 練習や対戦準備を行う選手
- 配信、デッキ、戦績の整理を担当するメンバー
- チーム内でメタゲームや対戦相手の傾向を確認するメンバー

一般公開の分析サービスではなく、チーム内の運用に合わせて必要な情報を素早く確認できる内部ツールとして扱います。

## 支援したい活動

- 選手ごとの配信活動量、配信媒体、配信内容の把握
- 配信アーカイブと使用デッキの紐づけ
- デッキ利用状況、クラス、アーキタイプの確認
- 試合結果、対戦相手、使用デッキ、勝敗の整理
- 大会や練習期間ごとの振り返り
- チーム内で共有すべき確認事項、未入力データ、判断待ち項目の可視化

このリポジトリは、収集、編集、確認、公開範囲の管理を一連の流れとして整備し、チームが同じ情報を見ながら議論できる状態を目指します。

## チーム内限定ツールとして扱う方針

PS_analyzer は、公開前提の情報発信サイトではなく、チーム内限定の分析ツールとして設計します。

そのため、今後の公開基盤は Cloudflare Access などの認証レイヤーを前提にし、閲覧者をチーム関係者に限定します。GitHub Pages の直URL公開や、認証されていない Save API の利用は、長期運用の前提にはしません。

限定公開化に伴い、次の方針を重視します。

- 閲覧できる人を明示的に制御する
- 保存や更新ができる経路を限定する
- 公開済みデータの棚卸しと無害化を行う
- 内部運用に必要なログ、手順、責務を残す
- セキュリティ強化と機能改善を別スコープとして扱う

## やらないこと

- 一般利用者向けの公開Webサービス化
- 不特定多数に向けたランキング、評価、批評コンテンツの提供
- チーム外部の利用を前提にしたアカウント管理機能の作り込み
- ソーシャル機能、コメント機能、公開プロフィール機能の追加
- 手元で管理すべき機密情報や個人トークンをブラウザに入力させる設計
- 目的が曖昧なデータ収集や、使い道のない生成物の保存

必要な機能を広げる場合も、チーム内の意思決定、対策、振り返りに直接つながるかを判断基準にします。

## 長期的な理想状態

長期的には、PS_analyzer をチームの分析作業の入口にします。

理想状態では、配信データ、デッキ情報、戦績情報が同じダッシュボードから確認でき、未入力や要確認のデータも明確に分かります。各メンバーは、最新の配信状況、使用デッキ、試合結果、期間別の傾向を迷わず確認できます。

運用面では、データ収集とダッシュボード生成が自動化され、編集や保存は認証された経路に限定されます。機能追加は issue 単位で進め、セキュリティ、公開範囲、分析価値を分けて判断できる状態を維持します。

PS_analyzer の最終的な価値は、単にデータを集めることではなく、チームが「今何を見るべきか」「次に何を確認すべきか」を素早く判断できることにあります。
